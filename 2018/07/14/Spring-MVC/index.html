Spring-MVC中的一些思想

IOC
1）核心servlet初始化的时候，对包中的类进行加载，并将类的全限定名放到set中
2）遍历类的全集set，找到标有指定注解的类，比如@Controller @Service，对这些类通过反射实例化，并放到map中key是类名 value是实例，这个map称为beanMap
3）遍历map，对每个类的属性进行遍历，如果属性有自动注入的注解(@resource @inject),那么对该类对应的实例通过反射进行当前属性的注入
4）经过第三步骤，map中的value就是已经注入成功的

URL Mapping
1）遍历类的全集set，找到标有@Controller注解的bean，放到新set中，称为Controller Set
2）遍历Controller Set中每个类的方法，找到标有@RequestMapping/@Inject的方法，将方法的value作为key，将封装有类和方法的对象(称为handler)作为value
3）每当有请求来到核心servlet，会对request进行解析，根据解析后的内容，找到对应的handler（包括class和method）
4）解析request中的参数，包括post请求和get请求
5）method.invoke(obj,args) 反射执行对应方法，这里的obj是根据class从beanMap中获取的已经注入的实例，可以放心调用
6）根据反射调用的结果，判断是jsp还是普通的json，reponse对二者的支持方式不同
